Table.h

Implementation 
+---------------------------------------------------------------+
Table.h provides a class for outputting data in a properly
formatted table, while abstracting away the string handling.
In addition, Tables can be used as menus from which a user can
select an item.

Also includes:
Strings.h
Getch.h
Debugging.h


Constructor:

Table(name="", width=0, height=0, seperator=" | ")


Methods:

void Add(...)
     - Add(vector<string>)
     - Add(size, string[])
     - Add(string, string)
     - Add(string, int/double/size_t)
    Appends a new row to the table.
    If you want more fine access to the table, it's just stored
    as a vector of vectors of strings called cells, and you can
    operate however you choose on that.

void show(showHeaders=false,showBorder=true,
              highlight=true,selX=-1,selY=-1)
    directly outputs toString(), and has identical arguments
    Used just for simplifying code.

Tuple<size_t,2> execMenu(showHeaders=false,
                             showBorder=true,
                             hightlight=true)
    (see DataStructures.h for Tuple)
    Begins a loop where the user can traverse the table with
    H,J,K,L keys (as in VIM) and select a cell.
    Returns a tuple like (x,y) of the selected cell.

void Set(x, y, string)
    sets the value of (x,y) in the table to string.
    Throws runTime error: Cannot set cell if (x,y) 
    is out of range.

string get(x,y)
    Returns value of cell (x,y)

string operator(x,y)
    Returns value of cell (x,y)
    Synonymous with get(x,y)
    Called like "TABLENAME(x,y)"

mlstring toString(showHeaders=false,showBorder=true,
                      highlight=true,selX=-1,selY=-1)
    (see Strings.h for mlstring)
    returns a multi-line string of the formatted table.
    showHeaders- only display row/collumn 0 (headers) if true
    showBorder - adds border if true (title will not be 
                 shown without border)
    highlight  - enables alternating highlighting of lines
    selX/selY  - generally not used except internally for
                 executing the menu. If either aren't -1, 
                 highlighting is disabled and cell(x,y) will
                 be highlighted instead.

void clear()
    resets table to all defaults
